# System Patterns: Green Code & Agentic Architecture

## 1. Green Code Standards (2026 Methodology)
All development must adhere to the **GreenServ** methodology to minimize energy leaks and embodied carbon[cite: 1, 2].

### Android (S24 Ultra) Patterns
*   **Aging-Aware Core Management**: Implement task scheduling prioritizing "Embodied Carbon Amortization," suspending background agents when renewable energy availability is low[cite: 2].
*   **Thermal-Aware UI**: Kotlin/Compose components must scale refresh rates and workload placement based on real-time ML-centric thermal prediction[cite: 1, 2].

### FastAPI/Backend Patterns
*   **GreenServ Dynamic Routing**: Route LLM inference tasks based on energy-performance tradeoffs and GPU/TPU scaling[cite: 1, 2].
*   **ARKV Management**: Implement Adaptive and Resource-Efficient KV Cache management to minimize memory pressure during long-context reasoning[cite: 2].

## 2. Agentic Integration Patterns
*   **Mission Control**: All agents operate asynchronously in dedicated workspaces to prevent context-switching fatigue[cite: 1, 2].
*   **Artifact-Based Transparency**: Every agent action MUST produce a verifiable deliverable (task lists, browser recordings) before being considered "usable software"[cite: 1].
*   **A2UI Protocol**: Use the JSON-based **Declarative Interface Protocol** for agent-generated UI to ensure security and prevent imperative code execution[cite: 1, 2].
